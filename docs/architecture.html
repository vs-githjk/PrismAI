The whole system at a glance

PrismAI is a meeting-intelligence app: a React frontend, a FastAPI backend gateway, and several subsystems that turn meeting input into structured intelligence — offline (paste/upload) and live (an in-meeting voice bot). Each box below is a self-contained area.

Overview › Analysis Pipeline

Analysis Pipeline — LangGraph, two tiers

A compiled StateGraph runs the 9 agents in two parallel tiers. Routing is deterministic (no LLM) — it always runs every agent, gating only sentiment to 2+ speakers. A barrier between the tiers builds a shared context so Tier 2 agents are enriched by Tier 1 results.

Overview › Live Bot & Voice

Live Bot & Voice

The bot joins a real meeting via Recall.ai. Two ways to look at it: (1) the internal voice I/O loop — speech in, reasoning, speech out; and (2) how the voice core wires into the rest of the system. Both are below.

1 · Voice I/O loop — STT → inference → TTS

Recall handles the audio bridge; everything between the transcript webhook and the spoken reply is ours.

Overview › Knowledge / RAG

Knowledge Base / RAG

Overview › Tools & Integrations

Tools & Integrations

A registry of callable tools. Both the live bot and the chat /agent path dispatch through execute_tool(), which enforces an owner-gate on side-effecting tools and routes to the right integration.

Owner-gate
Side-effecting tools (send email, post Slack…) only fire for the confirmed bot owner.
Confirmation
In a live meeting, the spoken command itself acts as the confirmation.
requires key
A tool only appears if its credential (e.g. google_access_token) is present.

Overview › Auth

Supabase issues the session token; require_user_id() validates it on gated endpoints. Google Calendar uses a separate PKCE flow because Supabase doesn't persist the provider token. A few demo endpoints are intentionally open.
