# PrismAI — Design Context

## Design Context

### Users
Engineering managers — in back-to-back meetings most of the day. They need to extract clarity from meeting chaos quickly. They value density, precision, and tools that do the work without ceremony. Sophisticated enough to notice good design, won't forgive slow or confusing UX. They open this tool between calls, not during leisure.

### Brand Personality
Premium, reliable, precise. The tool you trust to handle it correctly — every time.

### Aesthetic Direction
Dark (#07040f base), premium, technically precise. Core visual identity: WebGL prism animation — refracted light as metaphor for one messy input split into structured intelligence. Spectrum rainbow for the logo. Accent palette: cyan, sky, teal. Fonts: Sora (logo/display), Inter (body/UI), Geist (available). Feels like expensive developer tooling — Vercel or Linear energy, but with more visual personality from the prism motif.

Anti-references: Stock-photo SaaS landing pages. Generic AI tool aesthetics with robot icons and neon gradients. Anything consumer-grade or soft.

### Design Principles
1. **Reliability over urgency** — Nothing feels rushed or hyped. Every element communicates confidence, not pressure.
2. **The prism as metaphor** — One messy input → many structured outputs. Design reinforces this transformation narrative.
3. **Specificity builds trust** — Sample outputs, agent names, exact numbers. Vagueness kills trust with engineering managers.
4. **Generous space, clear hierarchy** — Room to breathe. Typography and spacing do the work; decoration earns its place.
5. **Dark and intentional** — Not dark because it's cool. Dark because this is a precision tool used during focused, professional work.
